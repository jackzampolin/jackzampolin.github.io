Arrays and Hashes in Ruby


A brief review


18 Feb 15

During week 3 at DBC we went back and looked at basic ruby syntax. A part of that was looking at hashes and arrays. Well arrays more than hashes. These two parts of ruby syntax are commonly used in many different applications. We will take these two methods of storing data and look at them one by one.

Arrays

          my_empty_array = [] || my_empty_array = Array.new
          foo_array = ['foo','foos','fouie','food','fool',2,2.05 ]
          array_of_arrays = [[2,3,4],['foo','food','foos'],[2.0,3.1,5.6]]

Above are three arrays. The first one is an empty array assigned to a variable. Empty arrays are useful if you plan on adding elements to it later in the program. For example; if you want to search a string for a particular pattern and push the sentence around what you are looking for to an array, you could start with an empty array. The second example above shows that you can have many different types of objects contained within a specific instance of an array. This means that you can even nest arrays within arrays as shown in the third example. This ability allows arrays to be extremely flexible

          puts foo_array[0] => 'foo'
          puts foo_array[2] => 'fouie'
          puts foo_array[-1] => 2.05 
          puts array_of_arrays[2][2] => 'food'

We have seen how to use arrays to store data, but how do we retrieve the data we have stored? Arrays, like strings, have the first element in the array named [0], the second [1] and so on. So if you need the third element out of foo_array you would type foo_array[2] as shown above. If you need to pull the last element out of an array you could see how long the array is and subtract one to get the index OR you can use [-1] which points to the last element in any array. [-2] would point to the second to last and so on. This is very useful if you have a constructed array of indeterminate length and need to access the last element. 

          array_of_arrays.flatten => [2,3,4,'foo','food','foos',2.0,3.1,5.6]
          ruby_array = ['c','e','g','f','b','d','a']
          ruby_array.sort => ['a','b','c','d','e','f','g'] 
          ruby_array.length => 7
          [[1.1,2.2,3.3],[1,2,3]].flatten.sort => [1,1.1,2,2.2,3,3.3]

The above code snippets show a couple of common array method calls and their results. The .flatten method takes an array of arrays and removes the interior arrays creating a single array. We applied it to our array_of_arrays from earlier. .sort takes the elements of the array and sorts them, either in alphabetical order or numerical order from either a..z or 1..100. If you need the array reversed you can apply the .reverse method! Another useful array method is .length. This method outputs the length of the array in elements as an integer. As you can see above we have called this method on our ruby_array yielding 7. The final example shows two methods called directly on an array, not just it's variable. The .sort method allows you to mix floats and integers, but not strings and fixnums.

Hashes

          dbc_hash = Hash.new || dbc_hash = {}
          my_info = {
            first_name: 'Jack',
            last_name: 'Zampolin',
            hometown: 'Richmond',
            age: 28,
       }
          puts my_info[:first_name] => 'Jack'
          my_info[:current_town] = 'Bozeman'
          puts my_info[:current_town] => 'Bozeman'

Above is shown the basic syntax for hashes in ruby. The first line shows the literal notation for initalizing a new hash. my_info shows an example hash. As you can see the individual elements in the hash are key-value pairs. So the :first_name refers to the value 'Jack'. Data is extracted from hashes by naming the key. You can see this at work in the 3rd line of the example. If you want to add a key-value pair to an existing hash the notation is shown in the fourth line. Hashes, like arrays, have methods to manipulate and view data within them. Hashes are mainly used in ruby to create and hold classes. But that topic is slightly beyond the remit of this post, but I have a feeling that we will cover classes in a later post. Stay tuned.
