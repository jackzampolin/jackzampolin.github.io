Classes and Modules in Ruby


The hows and whys


11 Mar 15

This week at DBC we continued to look at Classes in ruby. One of our textbooks is The Well Grounded Rubyist and while reading through the Class chapter I noticed a number of references to how similar Modules are. Learning that took me through the next chapter, conveniently all about the topic. Modules are much like Classes in that they have instance methods. This is where the similarities end. Classes can be used to store data while Modules can only store methods or constants. Modules must be included before they can be used. Below is some basic syntax to set up a Module and use one of its instance methods.

          module Cryptonomicon
            def bobby_shaftoe
              p "I see you and yours have displayed adaptability."
            end
          end
           
          class Manila
            include Cryptonomicon
          end
           
          dm_shaftoe = Cryptonomicon.new
          dm_shaftoe.bobby_shaftoe => "I see you and yours have displayed adaptability."

A major difference between Modules and Classes is that Modules do not have instances. calculus = Math.new results in a NoMethodError. To be used Modules must be included, as shown in the above example, in the class or program that plans to use them. Modules can store Classes defined within them however. 

          module Neuromancer
            class Sprawl
              def bama
                p "Boston-Atlanta Metropolitan Axis"
              end
            end
          end
           
          molly_millions = Neuromancer::Sprawl.new
          molly_millions.bama => "Boston-Atlanta Metropolitan Axis"

This is just some of the basic syntax for Modules. There are many more tricks to be learned. Once common module is Math. It contains a number of trigonometric operators and constants for pi and e.
