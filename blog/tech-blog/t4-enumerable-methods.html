Enumerable#cycle


Enumerators


27 Feb 15

This week at DBC we continued to dive deeper into ruby and specifically the special methods that allow rubists to write elegant and readable code. Some of these methods come from the Enumerable module. Enumerable includes some cool methods like .sort_by and .include?. Another great enumerable function is called .cycle

          a = ["a", "b", "c"]
          a.cycle { |x| puts x }  # print, a, b, c, a, b, c,.. forever.
          a.cycle(2) { |x| puts x }  # print, a, b, c, a, b, c.

At first it seems like just another way of iterating through arrays or hashes a specified number of times. And this method is extremely useful in that way. There is another way to use .cycle however.

          a = ["a", "b", "c"]
          b = a.cycle => Enumerator: ["a", "b", "c"]:cycle 
          b.inspect => Enumerator: [\"a\", \"b\", \"c\"]:cycle
          b.next => 'a'
          b.next => 'b'
          b.next => 'c'
          b.next => 'a'
          b.rewind => Enumerator: ["a", "b", "c"]:cycle
          b.next => 'a'

.cycle, among other methods allows you to create an Enumerator object. The enumerator object b in the example above illustrates that. b stores the array and a position, initially 0. You can use the enumerator methods to manipulate the object in a number of ways. .next returns the next element in the array or hash. If you want to reset the index to the beginning, utilize the .rewind method.

          b.next_values => ['a']
          b.next_values => ['b']
          b.next_values => ['c']
          b.peek => 'a'
          b.peek_values => ['a']
          b.size => 3

Shown above are four enumerator methods: .next_values, .peek, .peek_values, and .size. .next_values works just like .next, but it returns the value as an array instead. .peek returns the current value without changing the index. And, following the pattern, .peek_values returns the value as an array without changing the index. .size gives the size of the enumerator as an integer.

Programs can also run enumerable methods on enumerator objects. Thats a tongue-twister, and slightly beyond my skills at this moment. .cycle and enumerable objects are very cool and useful parts of ruby!
