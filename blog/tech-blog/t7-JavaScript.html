JavaScript Objects and Constructors


A foray into .js


21 Mar 15

For the last couple of weeks we looked extensively at Classes in Ruby. This week we changed things up and started looking into JavaScript syntax. It turns out there are constructs in JS that are closely analogous to Hashes and Classes in Ruby. JS calls these Objects and Object Constructors.

          var object = {};
          object.name = "Winston Churchill";
          object['age'] = 140;
          console.log(object); => { name: 'Winston Churchill', age: 140 }
          object.offices = ["First Lord of the Admiralty", "Prime Minister"];
          object.finestHour = function () { console.log("This was their finest hour.") }
          object.finestHour(); => "This was their finest hour."

Above is a sampling of the things you can do with a JS object. Line one illustrates Object Literal notation. Note that it is strikingly similar to initializing a hash in Ruby. Lines two and three show two different methods of adding data to new properties in the object. Line two is similar to the 'getter' and 'setter' functions provided by the attr_accessor in ruby classes, while line three is similar to adding a new key/value pair to a hash in ruby. JS objects can also be used to store arrays, other objects, or ( shown in line six ) functions. These functions are similar to instance methods in ruby. But say you need to create more than one object. Like a bunch of objects with similar properties. In ruby you would use classes but in JS you use Object constructors.

          function leader (name, age, offices, speech) {
            this.name = name;
            this.age = age;
            this.offices = offices;
            this.speech = function () { console.log(speech) } }
          var jfk = new leader ("John F Kennedy", 98, ['POTUS'], "Ich bin ein Berliner")
          jfk.speech(); => "Ich bin ein Berliner"

Even the spacing in the Object constructor function above shows the similarity between Ruby and JS. However where the two languages do differ is in the ability to access and change the data in objects from outside them. JS clearly makes this quite easy, while Ruby does a bit more to protect the stored data from outside encroachment. Unless you use the attr family of functions in Ruby there is not a great way to change or access data stored in instance variables with in Class objects. This is by design. Ruby also provides the #private method to further firewall methods and data stored in classes. This is just the briefest overview of the differences between these aspects of the two languages.
