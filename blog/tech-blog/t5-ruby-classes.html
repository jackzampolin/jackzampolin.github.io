Classes in Ruby


Deep in the weeds


7 Mar 15

One powerful aspect of the Ruby language is the ability to create your own classes. Classes are objects (like everything else in ruby) that contain methods. String, Integer and even Object are all different classes in ruby. I know this presents the obvious chicken and egg problem of all classes being objects, and Object being a class. This link has a good explanation of that conundrum. As it turns out Classes can be easily used to store data and model real world things.

What if you want an Object that models the classes a student is taking in school. You might want their name and then a list of the classes they are enrolled in and a way to add and remove classes

          class Student
            attr_accessor :name
            attr_reader classes
            def initialize ( name , classes=[] )
              @name = name
              @classes = classes
            end
            def add_class(name)
              @classes << name
            end
            def remove_class(name)
              @classes.delete(name)
            end
          end
          tom = Student.new( "Thomas Hardy" )

The above code sets this class of objects up for us. The word class is used to define the new class. The name of the class immediately follows: Student. The first method initialize defines what arguments you need to pass to create a new instance of the class. In the above example we set the variable classes to an empty array by default. This means that when creating new classes you can choose if you want to pass that argument or not. Under the initialize we set up our instance variables. Instance variables store information for specific instances of that class. So each Student we set up can have different values for the @name and @classes variables. Below that we define two instance methods to add and remove classes from our classes instance variables. The last line shows the syntax to create a new instance of the class. The attr_accessor call at the top of the method definition allows you to easily read and write into the instance variables without writing methods to do so. attr_reader allows just the easy reading of data from instance variables. The methods created by those calls are tom#name, tom#classes, and tom#name=. Those will be shown in action below.

          tom = Student.new( "Thomas Hardy" )
          tom.name => 'Thomas Hardy'
          tom.name = 'Bill Pullman'
          tom.name => 'Bill Pullman'
          tom.classes => []
          tom.add_class('Under the Greenwood Tree')
          tom.add_class('Far from the Madding Crowd')
          tom.add_class('Two on a Tower')
          tom.classes => ['Under the Greenwood Tree','Far from the Madding Crowd','Two on a Tower']
          tom => Student:0x007ff11588b018 @name="Thomas Hardy", @classes=["Under the Greenwood Tree", "Far from the Madding Crowd", "Two on a Tower"]
          tom.remove_class('Under the Greenwood Tree')
          tom.classes => ['Two on a Tower','Far from the Madding Crowd']

Above you can see the proper syntax to utilize all of the methods we set up for our Student class above. The Student#name and Student#classes instance methods allow for easy access of the data stored in the @name and @class variables respectively. The Student#name= method provided by the attr_accessor allows us to easily change the value of the instance variable as shown in line 3. Using our Student#add_class method we add a couple of classes to his list. Student#remove_class does the opposite. While we could have used the attr_accessor methods for this function, keeping the data set as an array would be more difficult. Our custom methods allow for that.

This is just some of the basic information on ruby Classes. There is much more to learn, but at least this is a starting point.
