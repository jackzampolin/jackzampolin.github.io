What is Rails?


Web Development frameworks


29 Mar 15

As phase 0 draws to a close an important question comes to mind. "I signed up for this class to learn how to do web programming in Ruby on Rails. Where is the Rails part?" We have seen quite a bit of Ruby but no Rails thus far. That begs the question, What is Rails?

According to rubyonrails.org (a site that would appear authoritative), "Rails is a web application development framework written in the Ruby language." Web application development frameworks are sets of libraries, templates, and databases designed to significantly reduce the time it takes to get a dynamic web application up and running from zero. Many of the tasks, such as setting up a Model Controller View (MCV) architecture, are automated in web dev frameworks.

Rails is not the only one of these, just one of the more successful. Large companies such as Twitter, Shopify, AirBnB, and Groupon built their sites using it. Rails emerged from work on a project called Basecamp in 2004 and has seen increasing adoption since then. The code base is actively updated and they continue to add new features and functionality. The most current version of Rails is 4.2. Rails inherits the guiding principle of Don't Repeat Yourself (DRY) programming from Ruby and also strongly favors convention in the naming and design of programs. Reading tutorials I have found the phrase "the Rails way" quite a bit. Theres even a book. Culturally this seems to be a carry over from Ruby where much of the same language is found.

Looking forward to phase 1, I am extremely excited to sink my teeth into rails in a more concrete way. Having only done the Hartl tutorial, which I highly recommend, digging deeper into this framework is something I am looking greatly forward to.
