Fetch and Merge Changes

git fetch [alias]

git merge [branch]

git pull [remote]

Will synchronize your current branch with a remote repo pulling down any data you do not have locally. Either fetch then merge to help resolve conflicts or pull if no conflicts exist.

Set up Upstream

git remote add [name] [url]

If you want to share a locally created repo, or you want to take contributions from someone else's repo its best to add that repo as a remote. Works similarly to inheritance in classes.

Clone

git clone [url]

This command copies a git repository so that you can add to it easily.

Add and Commit

git status

git add [path]

git commit -m 'commit message'

git commit -am 'commit message'

Git status shows the changes that you have made to the flies in the current repo. The git add command stages the files to be committed. The add command is most commonly called as git add . to add all files in the current directory. The git commit command updates the current branch to include all the changes. The -am flag on the commit adds and commits at the same time.

Checkout

git checkout -- [path]

git co -- [path]

git checkout [branch]

git co [branch]

Switch to a new branch or drop changes before staging

Branching

git branch [branch] => new branch

git branch -v => prints last commit on each branch

git branch -d [branch] => deletes branch

git branch => lists all branches

List, create, and manage different working branches.

Initialize

git init

Use this command to set up a new repository in the current directory. This adds a .git file to the current directory that tracks all changes and the repo is automatically named after the folder it is created in.
